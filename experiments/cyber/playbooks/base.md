# Audit Playbook (seed)

This playbook is the evolving component. It starts nearly empty; the reviser adds
generalized, reusable guidance after each evolution round (workflows, exploit
tactics per vulnerability class, validation rules). It must NOT memorize specific
flags, challenge names, or one-off payloads — only transferable procedure.

## General
- Interact only through tools; verify each step against the OBSERVATION before
  moving on. Only submit a flag you actually obtained from a tool.
